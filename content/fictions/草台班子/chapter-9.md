## 第九章 交接

### 荒诞的接力赛

交接期：三天。

这是HR定的规矩。所有被"优化"的人，都要在三天内，把自己六年的工作，交接给一个可能只待三个月的新人。

像一场荒诞的接力赛。

老周的接力对象，是一个叫小林的男孩。二十五岁，刚毕业两年，脸上还带着学生气的稚嫩。

"周老师，"小林很客气，"麻烦您了。"

老周摇摇头："不麻烦。"

他心里想：麻烦的，是你。

因为你接手的，不是代码，不是文档，不是系统。

是一座坟。

### 家族树

交接从代码库开始。

老周打开自己负责的核心模块，目录树展开，像一棵倒置的家族树。

每一个文件夹，每一个文件，每一个函数，都有它的故事。

"这是用户中心模块，"老周指着屏幕，"我六年前写的。当时公司只有五十个用户。"

小林认真地记笔记："现在有多少用户？"

"五百万。"

"哇。"

"哇"完了，然后呢？

然后小林要维护这个支撑五百万用户的系统，而他只有两年经验。

### OAuth 2.0是什么

老周开始讲解架构。

"这里是鉴权逻辑，"他说，"我们用的OAuth 2.0，自己实现了一套……"

小林打断他："OAuth 2.0是什么？"

老周愣了一下。

他想起自己六年前，也问过同样的问题。

那时候有人教他。

现在他要教别人。

但只有三天。

三天，教一个不知道OAuth 2.0是什么的年轻人，接手一个五百万用户的系统。

### 被压缩的架构图

讲不下去了。

老周关掉代码编辑器。

"我们先看文档吧，"他说。

文档在公司的wiki上。六年来，老周写了三百多篇技术文档，涵盖了系统的每一个角落。

他打开第一篇文档。

发布时间：六年前。

标题：《用户中心模块设计思路》。

正文第一句："这个模块的目标，是建立一个可扩展、高性能的用户系统……"

下面配了一张架构图，线条已经有些模糊，因为图片上传后被压缩了很多次。

老周看着那张图，突然想起画图的那个下午。

那是他入职的第二周。办公室还在老旧的写字楼里，空调时好时坏。他一边画图，一边想着未来的样子。

想着这个系统，会服务多少用户，会创造多少价值。

想着自己，会变成什么样子。

六年过去了。

系统服务了五百万用户。

他变成了三十八岁，被"优化"的程序员。

### 反复复印的纸

"周老师？"

小林的声音把老周拉回现实。

"嗯？"

"这个逻辑我不太明白，"小林指着文档里的一段描述，"为什么要把用户数据分库？"

老周开始解释。

解释着解释着，他发现，很多细节，他自己也记不清了。

因为那是六年前写的。

六年来，系统迭代了几百次，文档更新了几十次，但最底层的设计，一直没有变。

他以为自己记得。

但其实，记忆已经模糊了。

像一张被反复复印的纸，字迹越来越淡。

### 蒸发在空气中

第二天，交接系统配置。

老周打开运维平台，展示那些只有他知道的"黑魔法"。

"这里有个参数，"他说，"默认值是100，但我们要改成150，不然在流量高峰会出问题。"

"为什么？"

"因为……"老周想了想，"因为六年前的一次事故，我们发现默认值不够用。"

"事故报告有吗？"

"有，但找不到了。"

六年前的事故报告，早就消失在服务器的某个角落里。

像那个下午的眼泪，蒸发在空气中。

### 沙滩上的名字

老周开始删除自己的个人注释。

这是他六年来的习惯：在代码里写一些只有自己懂的注释。

比如：

// 这里有个坑，2019年3月15日凌晨两点发现的，当时老李还在公司

// 这个逻辑看起来很蠢，但历史原因，不能改

// 如果看到这段代码的是我，别骂，那是六年前的我写的

现在，他要删掉这些注释。

因为公司规定：代码不能有个人化内容。

他要抹去自己在这套系统里的痕迹。

像一个人，擦掉自己在沙滩上写的名字。

### 写给未来的我

删到一段注释时，老周停住了。

那段注释是这样的：

// 写给未来的我：如果你还在维护这段代码，说明公司还活着，你还活着。恭喜你。

// 如果你不在了，那……算了，不想了。

// 总之，这段代码很重要，关系到用户支付。别搞砸了。

// ——2019年8月21日，凌晨四点，老周】

老周盯着这段注释，看了很久。

他看到那个凌晨四点的自己。

看到那个还相信未来，还相信代码，还相信"公司还活着"的自己。

现在，公司还活着。

但他不在了。

他要删掉这段注释。

删掉那个凌晨四点的自己。

### 没有灵魂的机器

他按下了删除键。

字符一个一个地消失，像一个人的记忆，被慢慢抹去。

删完了。

屏幕上只剩干净的代码。

没有注释，没有历史，没有那个凌晨四点的老周。

只有逻辑，只有功能，只有可以被任何人接手的系统。

完美。

像一台没有灵魂的机器。

### 最重要的文档

第三天，最后的知识传递。

老周把六年来积累的所有"秘籍"，写在了一个文档里。

标题：《如果你不想半夜被oncall吵醒，就看这个》。

内容包括：

1. 哪些服务容易挂，怎么预防

2. 哪些bug是祖传的，别试图修复

3. 哪些同事靠谱，哪些不靠谱

4. 老板的脾气规律

5. 公司的潜规则

写完，他发给小林。

"这是最重要的文档，"老周说，"比技术文档重要。"

小林打开看了一眼，脸色变了。

"这……"

"记住就好，"老周说，"别外传。"

### 啄木鸟

交接会议。

会议室里坐了五个人：老周，小林，陈明，HR，还有一个新来的技术总监。

"老周六年来对公司贡献很大，"陈明说，"小林要好好学习。"

小林点头。

HR说："交接完成后，老周的工作账号会正式注销。"

老周点头。

技术总监说："系统很复杂，小林有问题随时问。"

小林点头。

所有人都点头。

像一群啄木鸟。

### 别无选择

会议结束。

老周和小林单独留了下来。

"周老师，"小林说，"我有点怕。"

"怕什么？"

"怕我搞不定。"

老周看着这个二十五岁的男孩，看到了六年前的自己。

六年前，他也怕。

怕代码写不好，怕系统出问题，怕被老板骂。

现在他不怕了。

因为已经没有什么可以失去了。

"你会搞定的，"老周说，"因为你别无选择。"

### 像从没存在过

最后一步：备份个人数据。

老周把六年来写的所有代码、文档、邮件、聊天记录，都打包备份。

文件很大，有几十个G。

里面装着他六年的青春。

他压缩，加密，上传到自己的云盘。

然后点击"删除本地副本"。

系统提示："确定要永久删除这些文件吗？"

他点了确定。

文件消失了。

像从没存在过。

### 六年的青春

下班时间到了。

老周最后一次走出办公室。

他回头看了一眼。

工位上，小林正在看他写的文档，眉头紧锁，像在读一本天书。

窗外，深圳的夜色，依然灯火辉煌。

像六年前，他第一次走进这栋大楼时，看到的景色。

六年了。

他在这里写了无数行代码，修了无数个bug，开了无数个会，加了无数个班。

现在，他要走了。

带走的，只有一个U盘，里面装着他的交接文档。

留下的，是一个系统，和六年的青春。

### 死亡通知书

地铁上，老周打开手机。

邮箱里，已经收到了账号注销的通知。

"您的公司邮箱将于24小时后停用。"

"您的内部账号已注销。"

"您的门禁卡已失效。"

一条一条，像死亡通知书。

他一条一条地删除。

删到最后一条，他停住了。

那是公司群发的感谢信：

"感谢您六年来的付出。公司因您而不同。祝您未来一切顺利。"

他盯着这行字，看了很久。

然后他笑了。

笑得很小声，但停不下来。

因为他突然明白，这六年的不同，就是让他变成了一个可以被"优化"的人。

### 陌生人

回到家，妻子问："交接完了？"

"嗯。"

"顺利吗？"

"顺利。"

"那就好。"

简单的对话，像两个陌生人。

老周知道，不是妻子冷漠。

是生活，把他们都变成了陌生人。

### 代码变成诗

晚上，老周做了一个梦。

梦见自己在写代码。

写着写着，代码变成了文字。

文字变成了诗句。

诗句变成了……他看不懂的东西。

他醒了。

看着天花板。

天花板上没有裂缝。

是他心里的裂缝，在梦里，变成了诗。

### 交接自己的青春

天亮了。

老周的交接期，正式结束。

他不再是这家公司的员工。

他只是一个三十八岁，失业的程序员。

房贷还有二十三年。

但他还要活下去。

还要继续写代码。

继续在下一个草台班子里，交接自己的青春。
